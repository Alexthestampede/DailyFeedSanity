  ---
  How to Push to GitHub Yourself

  It's actually pretty simple! Here's the basic workflow:

  Basic 3-Step Process:

  # 1. Stage your changes (add files to be committed)
  git add -A

  # 2. Commit with a message describing what you did
  git commit -m "Your commit message here"

  # 3. Push to GitHub
  git push

  That's it! git push by itself works because your repository already has
  origin configured as the remote.

  ---
  More Detailed Workflow:

  # Optional: Check what changed
  git status

  # Stage specific files (or use -A for all)
  git add dailyfeedsanity.sh
  # OR stage everything:
  git add -A

  # Check what's staged
  git status

  # Commit with a descriptive message
  git commit -m "Add launcher script"

  # Push to GitHub
  git push

  ---
  Quick Commands:

  # See what you changed:
  git status
  git diff

  # Stage + commit + push in one go:
  git add -A && git commit -m "Your message" && git push

  # See recent commits:
  git log --oneline -5

  # Undo unstaged changes:
  git restore filename.py

  ---
  Tips:

  - Good commit messages are short but descriptive: "Add feature X" not
  "stuff"
  - git add -A stages everything (new files, modifications, deletions)
  - git push defaults to git push origin master (your current branch)
  - Run from project directory - git commands work anywhere inside the repo
